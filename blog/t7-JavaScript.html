Ruby vs. Javascript

Programming languages tend to be similar, but no two are ever alike. Here, I'll demonstrate some of the key differences between Ruby and Javascript.

Syntax

One of the first things you'll notice is how different the syntax is between the two. For example: in Ruby, you use two equal signs to determine if two objects are the exact same object. In Javascript, however, three is used: 2 == 2 vs. 2 === 2. Javascript does have its own double equals sign, but that only determines whether the two values match, not if they're equal.

Another slight difference is incrementing. In both Ruby and Javascript, you can use x += 1 to increment by one. In Javascript though, there is a shorter method that you can use as well: ++ or -- (x++ increments by 1).

Methods and Functions

Functions in Javascript are the same as methods in Ruby. Again, the syntax is different:

Ruby:

    def hello
        puts "hello"
    end

Javascript:

    function hello() {
        console.log("hello");
    }

In Javascript, semicolons are used to mark the end of a statement. They are generally required, whereas in Ruby they are not. Also, the "console.log" is equivalent to "puts".

Hashes and Objects

Objects in Javascript are comparable to hashes in Ruby. They are used to organize key/value pairs:

    cat = {
        type :Scottish Fold,
        name :Meredith,         color :grey and white     }

    var cat = {
        type: "Scottish Fold";
        name: "Meredith";         color: "grey and white"     }

In order to access a hash in Ruby, you'd write cat[name] => "Meredith". In Javascript, you just call cat.name => "Meredith". Also, notice how you have to declare the cat variable in Javascript with the word var.

So I hope this was helpful in highlighting some of the important differences between Ruby and Javascript!
