Ruby Classes

Why use classes? Classes are the basic template from which object instances are created. Let's say you wanted to make a jukebox witha bunch of songs. We can set up a Ruby class that represents things that are songs. Classes are defined using the class keyword followed by a name, which must begin with a capital letter. For example:

Class Song
end

Whenever Ruby creates a new object, it looks for a method named initialize and executes it, since a blank object is not very exciting:

class Song
  def initialize(name, artist)
    @name = name
    @artist = artist
  end
end

For class Song, the initialize method takes 2 parameters: the name of the song and the artist. These just act like local variables within the method.

Since each Song object needs to carry around its own song name and artist, we need to store these values as instance variables within the object. An instance variable is simply a name preceded by the "@" sign. In this case, the parameter name is assigned to the instance variable @name and the same goes for artist.

Classes are never closed: you can always add more methods to an existing class. Let's say you wanted to print the details:

class Song
  def initialize(title, artist)
    @title = title
    @artist = artist
  end

  def details
    puts "#{title}" is playing by #{artist}.
  end
end

So now if you called the class = Song.new("Out of the Woods", "Taylor Swift"), that would print:

  "Shake it Off" is playing by Taylor Swift.

Classes are useful if you want to build objects without having to repeat code. You can view it as sort of a blueprint for a working application. I hope this simple tutorial helped you understand classes a bit better; till next time!
