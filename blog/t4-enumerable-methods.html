Enumerable #map

Ah, enumerables. Wait what now? Yes I feel the exact same way. I'm here to teach you a little something about enumerables, and in the process I hope you can get less confused with me.

Basically, an enumerable is just a bunch of methods packaged together that gets "mixed in" with other classes, like Arrays and Hashes. That means that a programmer can just write a method once, package them up as Enumerable, and tell Array or Hash to include them. Let's take a look at the #map enumerable as an example.

#map (also known as #collect), returns a new array by performing some action on every item in the given collection:

[1, 2, 3, 4].map {|x| x ** 2}
=> [1, 4, 9, 16]

["Aqua", "Bat", "Super", "Wonder Wo"].map {|adjective| adjective + "man"}
=> ["Aquaman", "Batman", "Superman", "Wonder Woman"]

When there is no block, the resulting array is the same as the given array:

[1, 2, 3, 4].map
=> [1, 2, 3, 4]

["Aqua", "Bat", "Super", "Wonder Wo"].map
=> ["Aqua", "Bat", "Super", "Wonder Wo"]

When #map is used on a hash, each key/value pair is passed to the block.

burgers = {"Big Mac" => 300, "Whopper with cheese" => 450, "Wendy's Double with cheese" => 320} burgers.map {|sandwich, calories| calories / 2}
=> [160, 150, 225]

When there is no block, #map returns an array containing a set of two-item arrays, one for each key/value pair:

burgers = {"Big Mac" => 300, "Whopper with cheese" => 450, "Wendy's Double with cheese" => 320}

burgers.map
=> [["Wendy's Double with cheese", 320], ["Big Mac", 300], ["Whopper with cheese", 450]]

So that is the Enuerable #map(or #collect) in a nutshell. Much better than having to use a for loop, amirite?
